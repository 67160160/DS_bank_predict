 ภาพรวมของโปรเจค 
โปรเจคนี้เป็นส่วนหนึ่งของวิชา Machine Learning Deployment โดยมีเป้าหมายเพื่อนำวงจรการทำ Data Science ตั้งแต่ต้นน้ำ (การทำความเข้าใจปัญหาและข้อมูล) ไปจนถึงปลายน้ำ (การนำโมเดลขึ้นใช้งานจริงบน Web Application) มาประยุกต์ใช้เพื่อแก้ปัญหาทางธุรกิจของธนาคาร

 การนิยามปัญหา 
ปัญหา: การทำแคมเปญการตลาดผ่านทางโทรศัพท์ (Telemarketing) แบบหว่านโทรหาลูกค้าทุกคน มีต้นทุนด้านเวลาและทรัพยากรบุคคลที่สูงมาก อีกทั้งยังสร้างความรำคาญให้กับลูกค้าที่ไม่มีความสนใจ
ทำไมต้องใช้ Machine Learning: ML สามารถวิเคราะห์รูปแบบ (Patterns) จากข้อมูลประชากรศาสตร์ สถานะทางการเงิน และประวัติการติดต่อในอดีต เพื่อทำนายว่าลูกค้ารายใดมี "แนวโน้มสูง" ที่จะตอบตกลงเปิดบัญชีเงินฝากประจำ ทำให้ธนาคารสามารถเลือกติดต่อเฉพาะกลุ่มเป้าหมายที่มีศักยภาพ (Targeted Marketing) ซึ่งช่วยลดต้นทุนและเพิ่มอัตราความสำเร็จของแคมเปญได้อย่างมีนัยสำคัญ

ข้อมูลที่ใช้ (Dataset)
ชุดข้อมูลที่ใช้คือ Bank Marketing Dataset (ไฟล์ bank.csv) ซึ่งประกอบด้วยข้อมูลของลูกค้าธนาคาร จำนวน 15 ตัวแปร (Features) โดยแบ่งเป็น 3 หมวดหมู่หลัก:

ข้อมูลประชากรศาสตร์: age, job, marital, education

สถานะทางการเงิน: balance, default, housing, loan

ประวัติการติดต่อแคมเปญ: contact, day, month, campaign, pdays, previous, poutcome

Target Variable: deposit (เปิดบัญชี = yes, ไม่เปิดบัญชี = no)

Business Insight (การจัดการ Data Leakage): ในโปรเจคนี้ได้ทำการ ตัดตัวแปร duration (ระยะเวลาการคุยสาย) ทิ้งก่อนการเทรนโมเดล เนื่องจากในสถานการณ์จริงพนักงานจะไม่มีทางทราบระยะเวลาการคุยสายล่วงหน้าก่อนทำการโทร การนำตัวแปรนี้มาใช้จะทำให้โมเดลเกิด Data Leakage และมีความแม่นยำเกินจริงแบบผิดปกติ

ขั้นตอนการพัฒนา (Methodology)
EDA & Data Preprocessing: ตรวจสอบการกระจายตัวของข้อมูล, กำจัด Outliers ในคอลัมน์ balance ด้วยวิธี IQR, และสร้าง Data Pipeline (ใช้ StandardScaler สำหรับตัวเลข และ OneHotEncoder สำหรับข้อมูลหมวดหมู่)

Model Development: ใช้ Random Forest Classifier โดยตั้งค่า class_weight='balanced' เพื่อจัดการปัญหา Imbalanced Data

Hyperparameter Tuning: ค้นหาพารามิเตอร์ที่ดีที่สุดผ่าน GridSearchCV พร้อมทำ 5-Fold Cross Validation

Evaluation: ประเมินผลโดยให้ความสำคัญกับค่า F1-Score, Precision และ Recall มากกว่า Accuracy เพื่อให้สอดคล้องกับผลกระทบทางธุรกิจ (การลด False Positive เพื่อประหยัดเวลาพนักงาน)

Deployment: พัฒนา Web Application ด้วย Streamlit เพื่อให้ผู้ใช้ทั่วไป (พนักงานธนาคาร) ใช้งานได้ง่าย
